# AGENTS.md

## Назначение

Этот файл задает правила работы AI-агентов в репозитории `spec-schema-org`.
Проект: простой статический сайт на Eleventy (фактически 2 страницы).

Приоритеты проекта:

- минимализм;
- качество кода.
- язык сайта: английский (весь пользовательский контент и интерфейс).

## Технологические решения (зафиксировано)

- Генератор сайта: Eleventy (11ty).
- Язык для конфигурации/утилит: TypeScript.
- Пакетный менеджер: `npm`.
- Node.js: `22.22.0` (ветка 22 LTS, фиксированная версия).
- Шаблоны: гибридный подход:
  - страницы с точной версткой: `Nunjucks` (`.njk`);
  - контентная страница: `Markdown` (`.md`) через `Nunjucks` layout.
- Стили: чистый CSS, без CSS-фреймворков на старте.

## Структура проекта

- Исходники: `src/`.
- Результат сборки: `_site/`.

## Правила для агента

- Не добавлять новые зависимости без явного запроса пользователя.
- Не усложнять архитектуру без необходимости.
- Сохранять решения простыми и поддерживаемыми.
- Не обновлять `README.md` автоматически (только если пользователь явно попросил).
- Деплой-цель: GitHub Pages с поддержкой двух URL: `https://spec-schema.org` (основной) и `https://anatoly-tenenev.github.io/spec-schema-org` (резервный).
- Если задача конфликтует с `AGENTS.md`, агент обязан явно запросить подтверждение у пользователя до выполнения.
- Если пользователь подтверждает правки, агент обязан обновить `AGENTS.md` в том же изменении, чтобы правила и реализация не расходились.

## Рабочий процесс Git

- Не работать напрямую в `main`.
- Всегда использовать отдельную ветку.
- Формат имен веток:
  - `feat/<kebab-case>`
  - `fix/<kebab-case>`
  - `chore/<kebab-case>`

## Обязательные команды

- Локальная разработка: `npm run dev`
- Сборка: `npm run build`
- Форматирование: `npm run format`

Перед финальной сдачей изменений агент обязан выполнить:

- `npm run format`
- `npm run build`

## CI (обязательно)

Для каждого PR должен быть включен GitHub Actions workflow с проверками:

- `npm run format -- --check`
- `npm run build`

В CI использовать Node.js `22.22.0`.
